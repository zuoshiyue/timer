# 训练间隔计时器APP - 替代调试方案

由于在Trae编译器中直接运行Gradle构建存在技术限制，以下是几种可行的替代方案：

## 方案一：使用Android Studio（推荐）

### 1. 安装Android Studio
1. 下载并安装最新版Android Studio
2. 安装必要的Android SDK和工具

### 2. 导入项目
```bash
# 在Android Studio中：
1. File → Open
2. 选择项目目录：/Users/lpf/Documents/docker/timer
3. 等待Gradle同步完成
```

### 3. 构建和运行
- **构建APK**：Build → Build Bundle(s) / APK(s) → Build APK(s)
- **运行调试**：Run → Run 'app'
- **设备选择**：连接Android设备或使用模拟器

## 方案二：使用命令行工具（需要安装Gradle）

### 1. 安装Gradle
```bash
# 使用Homebrew安装
brew install gradle

# 验证安装
gradle --version
```

### 2. 构建项目
```bash
cd /Users/lpf/Documents/docker/timer

# 清理项目
gradle clean

# 构建调试版本
gradle assembleDebug

# 构建发布版本
gradle assembleRelease
```

### 3. 安装APK到设备
```bash
# 连接Android设备后
adb install app/build/outputs/apk/debug/app-debug.apk
```

## 方案三：使用在线构建服务

### 1. GitHub Actions（如果项目在GitHub上）
创建`.github/workflows/android.yml`：
```yaml
name: Android CI

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v2
    - name: Set up JDK 11
      uses: actions/setup-java@v2
      with:
        java-version: '11'
        distribution: 'adopt'
    - name: Build with Gradle
      run: ./gradlew build
```

### 2. 其他在线服务
- **Bitrise** - 专业的移动应用CI/CD
- **CircleCI** - 支持Android构建
- **GitLab CI** - 自托管选项
